About the service

Pokemon service is a web service created by Kacper Koźmiński as part of a portfolio project. The service was made using Spring Boot with Java 17 and Gradle.

Technologies used in the project:

- Spring Security
- REST API
- Thymeleaf
- Bootstrap
- Spring Data JPA
- PostgreSQL

Service uses Database authentication with remember-me feature, providing basic html forms. Authentication credentials are verified for errors using custom & Spring validators. Authentication uses Roles and Authorities. Each logged user can change his password through an additional form and delete his account if he wants.

Service provides API that allows to display pokemons and their parameters, i.e. Pokedex number, full name, types and possible evolutions in table. Table rows can be sorted by ascending and descending order after each field except evolution. API supports pagination and can display 20, 40 or 60 pokemons per page. Additionally, each pokemon in the table can be displayed with details and its own photo by clicking on the table row.

The service enables interactions for logged users through the functionality of adding one pokemon of their choice to favorites. This feature is associated with the constantly updated Pokemon ranking, which is a table of 20 units with the most likes.

Pages are written in html with thymeleaf engine. Style of the pages was designed based on Bootstrap 5. Each image used on the website, except for the photos of the pokemons themselves, was generated by the AI text-to-image system DALLE 2. Pokemon pictures are official pokemon artworks used under a Fair Use Policy. They were provided to service via website pokemondb.net.
